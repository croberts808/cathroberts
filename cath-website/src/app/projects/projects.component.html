Flower Shop

Last December, I started a Google course through Coursera to learn more about UX Design and grow my skills as a designer and researcher. . . .

There are 7 courses total, and I am currently working my way through the 4th, “Conduct UX Research and Test Early Concepts.” In the first 3 courses, I learned all about user personas, user pain points, wireframing, prototyping, empathy, accessibility, and how to create user journeys and competitive audits.

In order to work through these courses, it was necessary to come up with an idea for a mobile or web application to put my newfound knowledge to work. I decided to create a floral delivery app that solves user pain points and is simple to use. Through the class, I learned how to use Figma to first create wireframes for the app, and then turn those wireframes into a prototype. See below for a walkthrough of my little app! What I Learned:

  • How to create digital wireframes from paper
  • How to use the Figma components tool
  • How to use Figma Variants to create interactive components
My Favorite Parts of the Entire Process:
  • Creating User Personas (see 'Alicia Wren' above)
  • Creating a User Journey Map to anticipate potential needs for the app
  • Learning how to navigate and use Figma

XL Check In System

This semester in my “Introduction to Software Engineering” course, we are practicing pair programming, learning Typescript, creating Angular projects, and discussing current events in the tech world. . . .

Although it is not my main goal to pursue a career in Software Engineering, I believe this course has been invaluable in teaching me how to work in teams and practice creating and deploying applications to the web without much assistance. Our most recent project has been to create a check in system for UNC's Computer Science Experience Lab.

The first rendition of this project required us to follow Angular tutorials to create the frontend of the website and deploy it to GitHub Pages. You can find the project at this link. I enjoyed learning about CSS and HTML for the first time, and spent additional time on the assignment researching how to change colors, add elements, create flex boxes, and create button events.

The second rendition of this project involved creating a backend to keep track of the form data entered in the frontend. We learned how to use the Observable class, HTTP methods, and FastAPI to complete this task. I decided to change the styling of this rendition to get extra practice with CSS, and also because I wanted to make it look nicer!

My Favorite Parts:

  • Working together with a partner
  • Choosing a color scheme that fit with the XL website
  • Using HTTP methods to retrieve data from the backend

Akari

During the spring 2022 semester, I created a JavaFX application for my “Foundations of Programming” course. We spent the semester creating programs in Java, and our final assignment was to learn how to use JavaFX and Maven to replicate the game Akari. . . .

It was my first time using the Model View Controller design pattern, and I ended up having a lot of fun creating the visuals and putting it all together. I also learned a lot about creating buttons, notifying observers, and using a build manager.

My Favorite Parts:

  • The trial and error of setting up a gridpane
  • Watching the model implementation work its magic
  • Showing the final product to my young nephew and allowing him to change the component colors with code
